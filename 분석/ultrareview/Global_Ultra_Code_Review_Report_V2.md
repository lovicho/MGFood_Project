## 🚨 심각한 논리적/구조적 결함 (Critical & Major)

### [UR-CRIT-01] PG 결제 웹훅 검증 로직 누락에 따른 결제 조작 취약성 (재강조 및 심화)
- **위치:** `외주업체제안/순대_ODM_온오프라인_쇼핑몰_구축_제안서.md` (3.2 온라인 결제 아키텍처)
- **메커니즘 (Why):** 1차 리뷰에서 지적한 바와 같이, 간편결제(네이버/카카오/토스페이) 연동 시 프론트엔드 콜백에 의존하는 구조는 치명적인 취약점입니다. 추가적으로, 주문 생성 트랜잭션과 PG사 승인 트랜잭션 간의 원자성(Atomicity)이 보장되지 않으면 재고가 차감되었으나 결제는 취소되는 "재고 불일치(Inventory Desync)" 상태가 발생할 수 있습니다.
- **수정 제안 (How):**
  - **Two-Phase Commit (2PC) 유사 패턴 적용**: 주문 테이블(Order)의 상태를 `PENDING` -> `AUTHORIZING` -> `PAID`로 세분화.
  - 서버 간(S2S) 검증 웹훅에 멱등성 키(Idempotency Key)를 필수 적용하여 중복 결제 승인 방지.
  - `Next.js` API 라우트 내부에서 데이터베이스 트랜잭션을 강제.

### [UR-CRIT-02] 오프라인 QR 결제 시 PWA의 상태 동기화 누락 위험
- **위치:** 제안서 5.1 오프라인 납품 연계 마케팅
- **메커니즘:** 오프라인 팝업 스토어나 제휴 매장에서 PWA 기반 QR 결제를 수행할 때, 사용자의 네트워크 환경이 불안정할 수 있습니다. 결제 완료 상태를 폴링(Polling)만으로 처리하면 네트워크 단절 시 고객의 앱 화면은 '결제 중'에 멈추지만 PG사 승인은 완료되는 데드락이 발생합니다.
- **수정 제안:** Server-Sent Events(SSE) 또는 WebSocket을 통해 서버에서 결제 상태 변경 이벤트를 PWA 클라이언트에 즉시 푸시하는 아키텍처를 `sw.js`(서비스 워커)와 통합하여 오프라인 재연결 시 상태를 동기화해야 합니다.

## ⚡ 리소스 및 성능 최적화 제안 (Minor & Optimization)

### [UR-OPT-01] 고해상도 AI 이미지 파이프라인(나노바나나)의 자동 변환/캐싱 아키텍처
- **위치:** `외주업체제안/나노바나나_AI_이미지_프롬프트_명세서.md` 연계
- **분석 & 개선안:** 생성된 8K 이미지를 수동으로 최적화하는 것은 운영 비용을 증가시킵니다.
  - **개선안**: Strapi CMS의 Media Library 플러그인 또는 AWS Lambda@Edge를 활용하여 이미지 업로드 시 실시간으로 WebP/AVIF로 포맷 변환 및 리사이징(모바일/태블릿/데스크탑용 3종)하는 On-the-fly Image Optimization 파이프라인을 구축해야 합니다.

### [UR-OPT-02] AEO/GEO 데이터 파싱 성능 병목 완화
- **위치:** 제안서 4. 발견 가능성 극대화 전략 (SEO / AEO / GEO)
- **분석 & 개선안:** SEO/GEO를 위한 JSON-LD 스키마가 서버 사이드 렌더링(SSR) 단계에서 매번 DB를 쿼리하여 생성될 경우 서버 리소스가 낭비됩니다.
  - **개선안**: Next.js의 `generateStaticParams`와 `ISR(Incremental Static Regeneration)`을 사용하여 제품 페이지와 공장 정보 스키마를 정적으로 빌드하고, 캐시 무효화(Cache Invalidation) 전략을 적용해야 응답 시간(TTFB)을 최적화할 수 있습니다.

## 🔍 아키텍처 및 코드 가독성 노트 (Information)

### 1. B2B / B2C 권한 분리(RBAC) 아키텍처 설계
- 제안서 내 '도매 주문 상담 폼'과 '일반 고객 밀키트 정기 배송' 기능이 혼재되어 있습니다. 데이터베이스 수준에서 사용자 역할(Role)을 `GUEST`, `B2C_USER`, `B2B_PARTNER`, `ADMIN`으로 명확히 구분하고, Next.js Middleware를 통한 라우트 보호(Route Protection) 가이드라인을 아키텍처에 추가 명시할 것을 권장합니다.

### 2. 크로스플랫폼 상태 관리 (State Management)
- 반응형 웹과 PWA의 복잡한 상태(장바구니, 결제 세션, 오프라인 모드)를 효과적으로 관리하기 위해 `Zustand` 또는 `Redux Toolkit`과 결합된 `React Query`를 사용하여 서버 상태와 클라이언트 상태를 완전히 분리하는 것이 유지보수성 측면에서 매우 유리합니다.
